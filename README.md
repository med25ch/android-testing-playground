# 🧪 Android Testing Playground

This repository contains my hands-on practice and experiments while learning **Android Unit Testing** using:

* Kotlin
* JUnit
* Mockito

The goal is to build strong testing skills and apply best practices to real-world Android applications.

---

## 📚 Course Reference

Based on the Pluralsight course:

> *Android Unit Testing with JUnit and Mockito Using Kotlin*

---

## 🎯 Objectives

* Understand the importance of unit testing in Android
* Write clean and maintainable unit tests
* Learn how to use **JUnit** effectively
* Master **Mockito** for mocking dependencies
* Test ViewModels, UseCases, and business logic
* Handle edge cases and error scenarios

---

## 🛠️ Tech Stack

* **Language:** Kotlin
* **Testing Framework:** JUnit 4 / JUnit 5
* **Mocking:** Mockito / Mockito-Kotlin
* **Coroutines Testing:** kotlinx-coroutines-test (optional)

---

## ✅ What I'm Practicing

* Basic unit tests (GIVEN / WHEN / THEN)
* Testing pure Kotlin classes
* Mocking dependencies with Mockito
* Testing ViewModels
* Testing coroutines and flows
* Error handling scenarios

---

## 🧪 Example Test Style

```kotlin
@Test
fun `GIVEN valid input WHEN login THEN return success`() {
    // GIVEN

    // WHEN

    // THEN
}
```

---

## 💡 Key Learnings

* Keep tests **independent and deterministic**
* Avoid testing implementation details
* Focus on **behavior, not code**
* Use mocking only when necessary
* Prefer **clean architecture** for testability
